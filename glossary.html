Glossary

Here you will find some key terms used on the Buddycloud website.

Buddycloud Channel

Read all about the Buddycloud channels here.

Buddycloud Protocol

Buddycloud is defined by a number of protocols: channel-server protocol, and the media server protocol (in progress). Buddycloud works with teams at the W3C, Mozilla Foundation, and XMPP Standards Foundation to champion the open, silo-free internet.

Cloud Install

The hosted version of Buddycloud.

Communication Silo

A communication silo refers to a service, like Skype for example, where users are unable to freely communicate with users on other services. Communication within a silo is always vertical, making it difficult or impossible for the system to work with unrelated systems. The opposite of a silo is a service like email where users on one email provider can email users on another email provider running different email software and the services interoperate to receive the message.

Content Plugins

The Buddycloud server supports validation of custom content types by means of a plugin system. By default the Buddycloud server supports Atom content. Additional content types can be supported by creating an appropriate validator and packaging it as a plugin.

Code

DNS Service Discovery

Buddycloud uses DNS to discover the API endpoint for any domain. DNS-SD returns a TXT record with the correct API to use. For example: query for the _buddycloud-api._tcp.example.com TXT record returns. "v=1.0 host=buddycloud.EXAMPLE.COM protocol=https path=/api port=443"

Extensible

Your personalised Buddycloud stack can run with just the components you need. Each Buddycloud component is designed to be extensible and easily re-deployable and avoids locking you into a one-size-fits-all messaging stack.


Federated (Distributed, Decentralised)

Examples of federated systems include the worldwide email network and DNS service. Each site can run its own email server and email from aunt@gmail.com reaches you@example.com. Just like email, all services on Buddycloud federate meaning that your users can connect, share and message with users on other Buddycloud-enabled sites. Without gatekeepers. Without limitations.


IoT (Internet of Things)

The network of physical objects embedded with electronics, software, sensors, and connectivity to create devices that exchange data.

On Premises

Buddycloud installed on your hardware, on your network and maintained by you.

Open Source

All Buddycloud code is released as open source on GitHub.

Code

Open Standards

Building on open standards means you tap into a rich set of existing libraries. Using open protocols also mean your code will interoperate with other services.

Push Notifications (Server Push)

Push notifications let you remotely wake up your application with a "there's new content" or "something happened" message. Sometimes this is just a "someone followed you" message. Other times it's a "you should pull down fresh content" command and your application can then synchronise the content.

Code

TLS Encryption

The Transport Layer Security (TLS) is a cryptographic protocol designed to provide communications security over a computer network. By default Buddycloud is setup to ensure all communication between clients and servers is encrypted.

XMPP

The Extensible Messaging and Presence Protocol (XMPP) is an open technology for real-time communication, which powers a wide range of applications including instant messaging, presence, multi-party chat, voice and video calls, collaboration, lightweight middleware, content syndication, and generalized routing of XML data.
